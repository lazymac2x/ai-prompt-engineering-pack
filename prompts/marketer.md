## 1. SEO Blog Article Generator

```
Write a comprehensive, SEO-optimized blog article.

Topic: {topic}
Primary keyword: {primary_keyword}
Secondary keywords: {keyword_2}, {keyword_3}, {keyword_4}
Target word count: {1500-3000}
Target audience: {audience_description}
Tone: {professional/conversational/authoritative/playful}
Competitor URLs ranking for this keyword: {url_1}, {url_2}

Requirements:
1. Create an outline first with H2/H3 heading structure
2. Include the primary keyword in: title, first paragraph, one H2, meta description
3. Write for humans first — no keyword stuffing
4. Include a compelling hook in the first 2 sentences
5. Add a "Key Takeaways" box at the top
6. Include internal link suggestions: {existing_content_topics}
7. End with a clear CTA: {desired_action}
8. Provide: meta title (under 60 chars), meta description (under 155 chars), URL slug
```

## 5. Landing Page Copy

```

Goal: {conversion_goal — e.g., "free trial signup", "purchase", "demo request"}

Current awareness level: {unaware/problem_aware/solution_aware/product_aware}
Price point: {price}
Competitors they're comparing us to: {competitor_1}, {competitor_2}

Write these sections in order:
1. Hero section: Headline (under 10 words), subheadline (1-2 sentences), CTA
2. Problem section: Agitate the pain (3 specific scenarios)
3. Solution section: Introduce the product as the answer
4. Features section: {number} key features, each with benefit-oriented heading + 2-line description
5. Social proof: Suggest types (testimonials, logos, metrics) and write 3 sample testimonials
6. Objection handling: Address the top 3 buying objections as FAQ
7. Final CTA section: Urgency-driven closing copy
8. Sticky header CTA text

For each section, note the persuasion principle at work (scarcity, authority, social proof, etc.)
```

## 9. A/B Test Design

```
Design an A/B test for {what_to_test — e.g., "email subject lines", "landing page headline", "CTA button color and text", "pricing page layout"}.

Context:
- Current metric: {metric_name} is at {current_value}
- Goal: Improve to {target_value}
- Monthly traffic/volume: {number}
- Tool we use: {optimizely/google_optimize/vwo/custom}

Please provide:
1. Hypothesis statement (format: "If we [change], then [metric] will [improve] because [reason]")
2. Control (A) description
3. Variant (B) description — and optionally (C) if a third variant is warranted
4. Primary metric to measure
5. Secondary/guardrail metrics
6. Minimum sample size calculation (show the math)
7. Expected test duration based on my traffic
8. Statistical significance threshold recommendation
9. Segmentation to analyze post-test (device, source, new vs. returning)
10. Decision framework: what results lead to what action
```
